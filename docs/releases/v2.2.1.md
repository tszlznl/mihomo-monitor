# v2.2.1

- 自动切换触发匹配修正
  - 自动切换不再只看 `chains[0]` 的实际节点名
  - 改为沿整条 `chains` 查找第一个已启用的自动切换策略组
  - 修复了“流量已经超过阈值，但因为命中的是节点而不是策略组名，导致不触发”的问题
- 自动切换面板默认值优化
  - 开启“自动切换”时，如果当前阈值为空、`0` 或旧默认值 `500`，自动填成 `100 MB`
  - 开启“自动切换”时，如果冷却时间为空或 `0`，自动填成 `10 分钟`
  - 开启“自动恢复原节点”时，如果静默恢复时间为空或 `0`，自动填成 `5 分钟`
- 子域名分组后的链路明细修正
  - 修复了主机分组开启后，`64.233.170.91` 这类 IP 形式主机在二级列表中可见，但右侧“连接明细”为空的问题
  - 当分组主机的 `primary` 本身是 IP 时，详情查询会把二级项按原始 host 精确匹配，不再误判成设备 IP

## 备注

- 本次发布主要修正自动切换与主机 drilldown 的判定细节，不改变已有 API 路径与配置结构。
