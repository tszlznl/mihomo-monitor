# v2.1.0

## Highlights

- 新增“大流量自动恢复”能力
  - 自动切换成功后，记录策略组当时的原始节点
  - 支持配置静默恢复时间，在后续若干分钟没有新的大流量触发时自动恢复
  - 恢复目标始终是第一次自动切换时记录的原始节点，不会被重复触发覆盖
- 自动切换事件记录增强
  - 新增 `restored`、`restore_skipped`、`restore_error` 三种恢复结果状态
  - 支持记录恢复取消原因和恢复失败原因，便于排查行为是否符合预期
  - 自动恢复检查结果与自动切换事件统一出现在最近记录列表里
- 自动切换配置面板升级
  - 新增“自动恢复原节点”开关
  - 新增“静默恢复时间（分钟）”输入项
  - 前端支持展示更清晰的恢复状态文案

## Behavior

- 自动恢复只会在成功自动切换后创建待恢复任务。
- 如果静默等待期间再次发生大流量触发，会刷新等待时间，不会丢失原始节点记录。
- 如果你手动修改了策略组当前节点，待恢复任务会自动取消，不会覆盖人工操作。
- 如果原始节点已经不在当前策略组选项中，会记录一次 `restore_error`，并清理该待恢复任务。

## Verification

- 已通过 `go test ./...`
- 已对真实 Mihomo Controller 做过端到端联调验证：
  - 自动切换成功后生成待恢复任务
  - 静默等待后自动恢复到原始节点
  - 手动改组后恢复任务被取消，并记录 `restore_skipped`
